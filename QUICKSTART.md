# 快速开始指南

## 🚀 5分钟快速体验

### 1. 访问应用

打开浏览器访问：**http://localhost:3004**

### 2. 登录系统

使用测试账号登录：

```
用户名: demo
密码: demo123
```

或者

```
用户名: admin
密码: admin123
```

也可以点击"没有账号？立即注册"创建新账号。

### 3. 打开文件管理

登录后，点击右上角的 **"管理文件"** 按钮，右侧会滑出文件管理面板。

### 4. 上传文件

1. 点击 **"上传文件"** 按钮
2. 选择一些测试文件（文本、图片、视频都可以）
3. 等待上传完成

### 5. 浏览和编辑文件

#### 文本文件编辑：
1. 点击任意文本文件（.txt, .js, .md 等）
2. 在 Monaco 编辑器中编辑内容
3. 按 `Cmd+S` (Mac) 或 `Ctrl+S` (Windows) 保存
4. 看到 "✓ 已保存" 提示

#### 图片预览：
1. 点击图片文件（.png, .jpg, .gif 等）
2. 在预览窗口查看图片

#### 视频播放：
1. 点击视频文件（.mp4, .webm 等）
2. 使用内置播放器播放视频

### 6. 创建文件夹

1. 点击 **"新建文件夹"** 按钮
2. 输入文件夹名称，如 "项目文档"
3. 在文件树中点击展开

### 7. 删除文件

1. 鼠标悬停在文件上
2. 点击右侧的垃圾桶图标
3. 确认删除

## 💡 测试建议

### 测试文本编辑：
1. 上传一个 `.js` 或 `.txt` 文件
2. 编辑内容，观察语法高亮
3. 保存文件，刷新页面看内容是否保存
4. 尝试快捷键 `Cmd+S` 保存

### 测试多媒体：
1. 上传一张图片（PNG/JPG）
2. 上传一个视频（MP4）
3. 点击查看预览效果

### 测试用户隔离：
1. 用 `demo` 账号上传一些文件
2. 退出登录
3. 用 `admin` 账号登录
4. 确认看不到 `demo` 的文件

### 测试文件夹：
1. 创建文件夹 "测试文件夹"
2. 上传文件到根目录
3. 注意：当前版本不支持上传到子文件夹（待实现）

## ⚡ 性能提示

- 单个文件建议不超过 **50MB**
- 视频文件建议使用 MP4 格式（兼容性最好）
- 图片推荐使用 WebP 格式（体积小）

## 🔧 开发者提示

### 查看用户文件

所有用户文件存储在：

```
file-manager/user-files/
├── user1/    # demo 用户的文件
├── user2/    # admin 用户的文件
└── user3/    # 新注册用户的文件
```

你可以手动在这里添加测试文件。

### 查看日志

服务器日志在终端中实时显示：
- 文件上传/下载
- 认证请求
- 错误信息

### 调试技巧

1. **打开浏览器开发者工具** (F12)
2. 查看 Network 标签页观察 API 请求
3. 查看 Console 标签页查看前端日志
4. 查看 Application > Cookies 查看 Token

## 🐛 常见问题

### Q: 无法上传文件？
A: 检查文件大小是否超过 50MB，检查浏览器控制台是否有错误。

### Q: 登录后看不到文件？
A: 首次登录，文件列表是空的。点击"上传文件"添加文件。

### Q: 编辑后刷新页面内容丢失？
A: 确保保存文件（Cmd+S），看到"✓ 已保存"提示后再刷新。

### Q: 视频无法播放？
A: 确保视频格式为 MP4/WebM，某些视频编码浏览器可能不支持。

### Q: 端口 3004 被占用？
A: Next.js 会自动选择其他可用端口，查看终端输出的实际端口。
